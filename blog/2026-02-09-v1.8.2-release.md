# ClawVault v1.8.2: OpenClaw Compatibility & Path Validation

**Release Date:** February 9, 2026

We're excited to announce ClawVault v1.8.2, focusing on OpenClaw compatibility and reliability improvements based on community feedback and automated code review.

## ЁЯОп What's New

### OpenClaw v2026.2.9 Compatibility

ClawVault is now fully tested and verified against OpenClaw's latest release (v2026.2.9). Key compatibility notes:

- **Hook system working** тАФ ClawVault's lifecycle hooks (`gateway:startup`, `command:new`, `session:start`) are fully operational after OpenClaw's tsdown migration fix
- **Post-compaction benefits** тАФ Session transcripts now maintain proper parentId chains, improving `session-recap` reliability
- **Memory improvements** тАФ Benefits from OpenClaw's Voyage embeddings and QMD cache optimizations

### OPENCLAW_HOME & OPENCLAW_STATE_DIR Support

For users with custom OpenClaw installations, ClawVault now respects:

```bash
# Custom OpenClaw home directory
export OPENCLAW_HOME=/custom/path/.openclaw

# Custom state directory (overrides OPENCLAW_HOME for agent data)
export OPENCLAW_STATE_DIR=/var/lib/openclaw
```

This enables:
- Multi-user deployments
- Containerized environments
- Custom storage locations

### Robust Path Validation

v1.8.2 adds strict validation for environment variable paths:

- **Whitespace trimmed** тАФ `"  /path  "` тЖТ `/path`
- **Empty strings ignored** тАФ Falls back to default paths
- **Absolute paths required** тАФ Relative paths are ignored to prevent cwd-dependent behavior
- **Graceful error handling** тАФ `listAgents()` now handles malformed filesystem state without crashing

## ЁЯФз Technical Details

### Path Resolution Changes

```typescript
// Before (v1.8.0)
export function getOpenClawDir(): string {
  const customHome = process.env.OPENCLAW_HOME;
  if (customHome) {
    return path.resolve(customHome);  // Could resolve relative paths
  }
  return path.join(os.homedir(), '.openclaw');
}

// After (v1.8.2)
function validateEnvPath(envValue: string | undefined): string | null {
  if (!envValue) return null;
  const trimmed = envValue.trim();
  if (!trimmed) return null;
  
  const resolved = path.resolve(trimmed);
  if (!path.isAbsolute(resolved)) return null;  // Reject relative paths
  
  return resolved;
}
```

### Error Handling Improvements

`listAgents()` now wraps filesystem operations in try/catch to handle edge cases:
- Non-directory at expected path
- Permission errors
- Corrupted state files
